# FleetOps Database Schema (`fleet.db`)

All data is **synthetic**, generated by [`src/generate_data.py`](../src/generate_data.py)
with `SEED = 42` and a fixed anchor date of **2026-07-08** — re-running the generator
produces a byte-identical database. Schema shape was inspired by public
predictive-maintenance repos (see PRD references); no real data was copied.

## Tables

### `trucks` (200 rows)
| Column | Type | Notes |
|---|---|---|
| `truck_id` | TEXT PK | `T-001` … `T-200` |
| `vin` | TEXT | synthetic 17-char VIN |
| `make`, `model` | TEXT | 8 make/model combos (Freightliner Cascadia, Volvo VNL 860, …) |
| `year` | INTEGER | 2016–2025 |
| `mileage` | INTEGER | current odometer |
| `status` | TEXT | `active` / `in_shop` / `retired` |
| `home_depot` | TEXT | 5 depots |
| `last_service_date` | TEXT (ISO) | |
| `next_service_due_date` | TEXT (ISO) | **past date ⇒ overdue** |
| `next_service_due_miles` | INTEGER | **below `mileage` ⇒ overdue** |

### `fault_codes` (12 rows)
SAE J1939-style codes (`SPN-xxxx-FMI-x`) with `description`, `severity`
(low/medium/high/critical), `system` (engine/aftertreatment/electrical/brakes),
`recommended_action`, and `related_part_id → parts`.

### `parts` (13 rows)
`part_id` (`P-001`…), `part_name`, `part_number`, `unit_cost`, `stock_qty`,
`min_stock` (**stock below min ⇒ reorder**), `supplier`, `lead_time_days`.

### `maintenance_log` (2,000 rows)
One row per maintenance event: `truck_id → trucks`, `event_date`, `event_type`
(`preventive`/`repair`/`inspection`/`fault`), nullable `fault_code → fault_codes`,
`description`, nullable `part_id → parts`, `labor_hours`, `cost` (parts + $135/hr labor),
`technician`, `odometer`.

## Seeded patterns (for evals & demo)

1. **19 overdue trucks** — active trucks past `next_service_due_date` and/or `next_service_due_miles`.
2. **5 problem trucks** — recurring `SPN-3226-FMI-4` (NOx sensor) faults, 4–6 each.
3. **2 low-stock parts** — DPF Filter Assembly (1 on hand, min 4) and NOx Sensor (3 on hand, min 6), both tied to common aftertreatment faults.
4. **Winter battery cluster** — ~70 `SPN-627-FMI-4` cold-start faults during Dec 2025–Feb 2026.
